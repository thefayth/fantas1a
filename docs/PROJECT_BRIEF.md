# Project Brief

## Project

Fantasia

## Canonical Repository

`thefayth/fantasia`

## One-Line Description

Fantasia is a local-first AI operations desktop for builders who need automation with receipts, rollback, review, and guarded publishing readiness.

## Why It Matters

AI work often sprawls across chat windows, local scripts, plugins, cloud dashboards, and half-remembered instructions. Fantasia gives that work a visible operating room: what is running, what is approved, what is blocked, what has a receipt, and what can be safely reversed.

## Who It Serves

- Builders running local-first AI operations.
- Website owners preparing visual and content updates.
- Operators who need review, receipts, rollback, and clear authority boundaries.
- Partners or investors evaluating practical AI workflow products.

## Current First Use Case

FVE website visual operations:

- review live website images
- keep passing images in place
- identify weak, repeated, broken, or line-drawn visuals
- prepare replacement candidates
- map candidates to slots
- check production readiness
- publish only through guarded workflows

## Public / Private Boundary

Public: project identity, docs, workflow diagrams, safe screenshots, brand notes, public status, and protected ownership posture.

Private: engine source, prompts, agent instructions, credentials, production adapters, private workflows, customer data, deployment systems, and operational infrastructure.
